# 大学生实习管理辅助平台 — 帮助文档

## 📋 系统概述

本系统是一个集**实习打卡管理、周报批阅、Agent智能分析、微信群推送**于一体的综合性实习管理辅助平台。系统采用 VS Code 风格界面，支持暗色/亮色主题切换，所有功能模块共享登录状态。

---

## 一、🔐 登录页面

### 1.1 登录方式

- **账号密码登录**：输入账号和密码，点击"登录"按钮
- **记住密码**：勾选后下次启动自动登录
- **自动登录**：如果之前保存了凭证，启动时会自动尝试登录

### 1.2 登录后欢迎页

登录成功后显示欢迎页面，包含：

- 当前登录账号信息
- 系统功能列表概览
- 使用提示
- **退出登录**按钮（点击后清除所有登录状态和缓存）

### 1.3 登录状态共享

- 登录成功后，**打卡管理、周报批阅、Agent智能体、推送报告**四个模块自动共享登录状态
- 无需在每个模块中重复登录

## 二、⏰ 实习打卡管理

### 2.1 前置操作

在顶部工具栏选择：

1. **角色**：选择当前身份（如学校二级管理员）
2. **届数**：选择学生届数（如2026届）
3. **学院**：选择所属学院
4. **班级**：选择目标班级
5. **围栏**：开启/关闭电子围栏预警（开启后会在日报中分析签到地点异常）

### 2.2 左侧功能按钮

#### ⏰ 时报组

| 按钮               | 功能说明                                                     |
| ------------------ | ------------------------------------------------------------ |
| **📋 实习打卡时报** | 生成当日实习打卡日报，包含：签到列表、周报数统计、地点异常分析、生成图片报告 |
| **▶️ 启动定时任务** | 启动整点定时报告生成（8:00-22:00），自动生成并保存日报       |
| **⏹️ 停止定时任务** | 停止正在运行的定时任务                                       |

#### ⚙️ 功能组

| 按钮               | 功能说明                                       |
| ------------------ | ---------------------------------------------- |
| **📤 分享报告**     | 打开日报图片保存目录（daily_practice_reports） |
| **🔔 一键提醒打卡** | 向当前班级未打卡学生发送系统内打卡提醒消息     |

#### 📊 报表组

| 按钮                   | 功能说明                                                 |
| ---------------------- | -------------------------------------------------------- |
| **📊 数据统计分析**     | 导出实习过程数据统计 Excel                               |
| **🚫 未实习学生统计**   | 导出未实习学生名单 Excel                                 |
| **📑 班级情况汇总**     | 导出班级实习情况汇总 Excel                               |
| **📝 自主实习计划管理** | 导出自主实习计划管理 Excel                               |
| **📋 统一实习计划管理** | 导出统一实习计划管理 Excel                               |
| **🏢 企业信息导出**     | 导出学生企业信息 Excel                                   |
| **📈 打卡汇总报表**     | 弹出日期选择对话框，导出指定日期的打卡汇总 Excel         |
| **📍 打卡地址异常分析** | 弹出月份选择对话框，导出指定月份的打卡地址异常分析 Excel |

### 2.3 数据表格

- 显示学生的签到状态、签到时间、签到地址、签退状态、签退时间、本周周报数等信息
- **右键点击学生**可进行单个学生的 Agent 智能分析（分析打卡异常行为）

### 2.4 自动推送

- 生成日报后，如果推送页面配置了"生成打卡时报后自动推送到微信群"，会自动执行推送

## 三、📝 周报批阅系统

### 3.1 前置操作

在顶部工具栏选择：

1. **角色**、**届数**、**学院**、**班级**（与打卡页面共享）
2. **开始/结束年月**：选择要查询的周报时间范围
3. **学生**：选择查询单个学生（默认"所有学生"查询全班）
4. **💾 保存配置**：保存当前顶部工具栏配置，下次启动自动加载

### 3.2 主要功能

#### 查询学生

- 点击 **查询学生** 按钮，根据顶部工具栏配置查询学生周报列表
- 支持按学院、班级、学生、时间范围筛选
- 查询结果包含学生姓名、学号、周报数、已批阅数、未批阅数等信息

#### 批阅操作

| 操作         | 说明                                   |
| ------------ | -------------------------------------- |
| **批阅**     | 选中学生，点击"批阅"按钮，逐篇批阅周报 |
| **批量批阅** | 选中多个学生，批量批阅周报             |
| **驳回**     | 将不合格的周报驳回给学生修改           |
| **追加评语** | 对已批阅的周报追加评语                 |
| **导出报表** | 导出周报批阅结果 Excel/CSV             |

#### 右键菜单

- 在学生表格中**右键点击学生**，可选择：
  - **🤖 AI批阅未批阅周报**：自动切换到 Agent 页面，使用 AI 批阅该学生的未批阅周报

### 3.3 自动批阅

- 系统支持自动批阅功能，可批量处理未批阅的周报

## 四、🤖 Agent 智能体

### 4.1 功能概述

Agent 智能体提供 AI 驱动的智能分析功能，包括：

- **学生签到行为分析**：分析学生打卡异常行为
- **周报质量分析**：分析学生周报质量
- **AI 周报批阅**：使用 AI 自动批阅未批阅周报

### 4.2 配置要求

使用 Agent 功能前，需在 **⚙ 齿轮按钮 → Agent配置** 中配置：

- **LLM API Key**：大语言模型 API 密钥
- **API 地址**：LLM 服务地址
- **模型名称**：使用的模型（如 deepseek-chat）

### 4.3 主要功能

#### 签到行为分析

- **单个学生分析**：在打卡页面的数据表格中右键点击学生，选择"单个同学Agent分析"
- **全班分析**：在 Agent 页面选择分析范围，点击分析按钮

#### 周报质量分析

- 分析学生周报的完整性、规范性、内容质量
- 生成分析报告并保存到 agent_reports 目录

#### AI 周报批阅

- **从周报页面触发**：在周报页面右键点击学生，选择"AI批阅未批阅周报"
- **在 Agent 页面执行**：Agent 页面会自动接收周报页面的缓存数据，执行 AI 批阅
- AI 批阅会生成评分和评语，并自动提交到系统

### 4.4 聊天界面

- Agent 页面提供聊天式交互界面
- 支持发送消息、查看分析结果
- 分析报告自动保存到 `agent_reports/` 目录

## 五、📤 推送报告到微信群

### 5.1 配置说明

在推送页面配置以下信息：

- **目标微信群**：输入要推送的微信群名称
- **微信程序路径**：微信可执行文件路径（留空自动查找）
- **推送内容选项**：
  - ✅ 推送实习打卡时报
  - ✅ 推送打卡地点分析
  - ✅ 推送 agent 打卡分析

### 5.2 推送操作

#### 手动推送

| 按钮                    | 功能说明                               |
| ----------------------- | -------------------------------------- |
| **📤 推送实习打卡时报**  | 推送最新生成的日报图片到微信群         |
| **📊 推送agent打卡分析** | 推送最新的 Agent 分析报告到微信群      |
| **🗺️ 推送打卡地点分析**  | 推送最新的签到地点异常分析图片到微信群 |

#### 定时推送

| 按钮               | 功能说明                                       |
| ------------------ | ---------------------------------------------- |
| **▶️ 启动定时推送** | 按配置的时间自动推送（默认 08:00,12:00,18:00） |
| **⏹️ 停止定时推送** | 停止定时推送任务                               |

#### 自动推送

- 勾选 **"生成打卡时报后自动推送到微信群"** 后，每次生成日报会自动推送到微信群

### 5.3 测试功能

| 按钮               | 功能说明               |
| ------------------ | ---------------------- |
| **🔗 测试微信连接** | 测试微信客户端连接状态 |
| **📂 打开报告目录** | 打开日报图片保存目录   |

### 5.4 推送内容说明

- 推送时会自动 **@所有人** 提醒群成员查看
- 推送内容根据三个配置开关决定：
  - 推送实习打卡时报（日报图片）
  - 推送打卡地点分析（异常地点分析图片）
  - 推送 agent 打卡分析（分析报告文本）

## 六、🎨 主题与设置

### 6.1 主题切换

- 点击左侧活动栏底部的 **☀️/🌙** 按钮切换暗色/亮色主题
- 主题切换会同步应用到所有页面

### 6.2 系统设置（⚙ 齿轮按钮）

点击左侧活动栏底部的 **⚙** 按钮，打开系统设置对话框：

#### 显示设置

- **字体大小调整**：拖动滑块或输入数值调整全局字体大小（10px - 24px）
- 实时预览效果

#### 加密配置

- **AES 密钥配置**：16位 AES 加密密钥和偏移量
- **RSA 公钥配置**：RSA 公钥（PEM 格式）

#### 高德API配置

- **API Key**：配置高德地图 Web 服务 API Key（用于签到地点解析）

#### 默认查询配置

- 编辑 app_config.json 中的各项默认配置参数

#### 关于

- 系统版本信息
- 功能列表

## 八、⚠️ 注意事项

1. **登录状态**：所有功能模块共享登录状态，退出登录会清除所有缓存
2. **定时任务**：启动定时任务后，程序需保持运行状态
3. **微信推送**：推送功能需要微信客户端已登录，且目标群聊存在
4. **Agent 分析**：需要配置有效的 LLM API Key
5. **高德API**：地点分析功能需要配置高德地图 API Key
6. **数据安全**：请妥善保管账号密码和 API Key 等敏感信息
